# S5 – Servidor de Almacenamiento (Storage)

## Objetivo de S5
Configuramos el servidor S5 utilizando una imagen de NGINX sobre Alpine para que actúe como un servidor de contenido estático especializado. Su única función es servir las imágenes que los usuarios han subido previamente a través de S4.

Al tener un servidor dedicado para las imágenes, liberamos a los nodos de aplicación (S2, S3) de la carga de procesar archivos binarios pesados, optimizando la velocidad de carga del feed de Extagram.

## Configuración y Funcionamiento de S5
- **Despliegue del Contenedor:** Este contenedor destaca por montar el volumen de imágenes en modo read-only (:ro). Esto es una medida de seguridad crítica: S5 puede leer y mostrar las fotos, pero no tiene permisos para borrarlas o modificarlas, protegiendo el contenido de posibles ataques externos.
  ```sql
  # Comando de ejecución manual
  docker run -d \
    --name s1-s1-s5-storage-1-1 \
    --network red_entrada \
    --network red_servicios \
    -v /home/ec2-user/sprint2/S5/storage:/usr/share/nginx/html:ro \
    nginx:alpine
  ```
- **Estructura del contenedor:**
  ```sql
  S5/
  └── storage/        # Carpeta física en el Host compartida con S4  
  ```
  | Fichero | Función Técnica |
  |--------|------|
  | /usr/share/nginx/html | Raíz del Servidor: Es el directorio interno de Nginx donde se sirven las imágenes. Está vinculado directamente al volumen de S4. |
  | nginx:alpine | Motor de Entrega: Imagen optimizada para servir archivos estáticos con el menor consumo de memoria posible. |
