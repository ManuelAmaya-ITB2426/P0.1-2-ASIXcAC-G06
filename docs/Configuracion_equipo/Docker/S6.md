# S6 – Servidor de Activos Estáticos (Static)

## Objetivo de S6
Configuramos el servidor S6 utilizando una imagen de NGINX sobre Alpine para centralizar todos los recursos que definen el aspecto visual de Extagram. Su función es servir archivos que no cambian con el uso de la aplicación, como la hoja de estilos CSS, logotipos, iconos y el archivo de previsualización SVG.

Al separar el diseño (S6) del contenido subido por los usuarios (S5), garantizamos una organización profesional de los activos y permitimos que el navegador cachee estos archivos de forma más eficiente.

## Configuración y Funcionamiento de S6
- **Despliegue del Contenedor:** Al igual que S5, este contenedor funciona en modo read-only (:ro), ya que su contenido es estático y solo debe ser modificado por el administrador del sistema al actualizar el diseño de la web.
  ```sql
  # Comando de ejecución manual
  docker run -d \
    --name s1-s1-s6-static-1-1 \
    --network red_entrada \
    --network red_servicios \
    -v /home/ec2-user/sprint2/S6/static:/usr/share/nginx/html:ro \
    nginx:alpine
  ```
- **Estructura del contenedor:**
  ```sql
  S6/
  └── static/
      ├── style.css      # Definición de colores, fuentes y layout
      └── preview.svg    # Icono predeterminado del formulario de subida    
  ```
  | Fichero | Función Técnica |
  |--------|------|
  | style.css | Hoja de Estilos: Define la estética "dark mode" de Extagram y el diseño responsivo. |
  | preview.svg | Asset Visual: Recurso gráfico ligero que sirve de guía visual en el formulario de publicación. |
